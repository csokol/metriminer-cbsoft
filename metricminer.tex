\begin{resumo} 
    Este trabalho apresenta uma ferramenta web que oferece suporte ao 
    pesquisador da área de mineração de repositórios de software, o MetricMiner.
    Esse sistema armazena diversas informações de repositórios e calcula uma 
    série de métricas de código fonte sobre todo o histórico dos projetos a partir
    dos seus sistemas de controle de versões. Posteriormente, o pesquisador é capaz
    de extrair os dados calculados pelo sistema e realizar análises 
    estatísticas sobre os resultados. 
    Como toda a interação é feita por meio de uma interface web, o MetricMiner 
    facilita muito o processo de pesquisa na área, já que com os softwares disponíveis
    atualmente, o pesquisador precisa configurar a ferramenta localmente e baixar os
    projetos sobre os quais deseja calcular suas métricas, tornando o processo difícil
    e muito demorado. Além disso, o sistema disponibiliza uma base com dados de uma
    série de projetos open source sobre os quais um pesquisador pode extrair e 
    relacionar informações, além de submeter novas métricas para serem calculadas para
    esses projetos. Também será apresentada uma análise simples de um projeto
    de grande porte, exibindo o potencial das funcionalidades da ferramenta.
\end{resumo}


\section{Motivação} % descrição e motivação do problema tratado pela ferramenta;

Minerar repositórios de software é uma tarefa custosa. Para desenvolver uma 
pesquisa na área, um pesquisador é obrigado a baixar diversos projetos
em sua estação de trabalho, realizar uma série de cálculos sobre o código
dos projetos e sobre os meta dados de seu repositório. Esse processo requer que
o pesquisador instale diversos softwares e bibliotecas localmente para reaproveitar
as ferramentas desenvolvidos nessa área, tornando o processo extremamente difícil 
para quem está iniciando nessa área de pesquisa.

Além de ser um processo complexo, esse tipo de pesquisa consome muitos recursos 
computacionais. Primeiro, é necessário baixar diversos repositórios a serem minerados
consumindo um volume considerável de banda. Depois, os dados devem ser processados e 
persistidos em um banco de dados, ocupando uma grande volume de disco, afinal
alguns projetos possuem um longo histórico de desenvolvimento que deve
ser armazenado localmente (algum exemplo?). Então, o pesquisador pode calcular métricas
sobre esses dados, além de extrair relações entre os meta dados do histórico do sistema de controle de versão,
gastando uma quantidade grande de processamento de CPU. Só depois de passar por todas
essas etapas, é possível extrair dados e validar hipóteses através de análises estatísticas.
